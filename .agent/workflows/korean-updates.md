# 코드 수정 지침

## 수정 전 필수 사항

1. **플랜 먼저 공유**: 코드를 수정하거나 새로운 기능을 추가하기 전에 반드시 계획을 먼저 설명합니다.
   - 무엇을 변경할 것인지
   - 왜 변경이 필요한지
   - 어떤 파일들이 영향을 받는지

2. **사용자 확인**: 복잡한 변경의 경우 사용자의 승인을 받은 후 진행합니다.

## 변경 설명 규칙

- 모든 변경 내용은 **한글**로 설명합니다.
- 변경 사항을 간결하고 명확하게 요약합니다.
- 기술 용어는 필요시 영어를 병기할 수 있습니다. (예: "컴포넌트(Component)")

## 예시 형식

```
📋 변경 계획:
- 파일: app/example.tsx
- 내용: 버튼 클릭 시 로딩 표시 추가
- 이유: 사용자 경험 개선

✅ 변경 완료:
- `handleClick` 함수에 로딩 상태 추가
- ActivityIndicator 컴포넌트 적용
```

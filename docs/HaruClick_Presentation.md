# 하루클릭 (HaruClick)

## 경조사, 이제 3초면 끝

청첩장 찍고, 금액 추천 받고, 일정까지 자동 등록

---

## 1. 우리가 푸는 문제

- 청첩장/부고장 받으면 먼저 드는 생각: `언제지? 얼마 내지?`
- 대부분 카톡, 캘린더, 메모장에 따로 기록
- 시간이 지나면 `누구에게 얼마 했는지` 기억이 흐려짐
- 결과적으로 경조사가 반복될수록 스트레스도 반복

### 한 줄 요약
`경조사 정보는 많고, 관리는 매번 처음부터` 입니다.

---

## 2. 해결 방법: 하루클릭

### 입력은 가볍게
- 사진 촬영
- 음성 한마디

### 처리는 똑똑하게
- AI가 날짜/장소/이름/금액 자동 추출
- 관계/지역/행사 기준으로 적정 금액 추천
- 공유 캘린더와 채팅으로 가족/팀 동시 관리

---

## 3. 사용자는 이렇게 씁니다

### 상황
"다음 주 결혼식인데, 일정도 등록해야 하고 축의금도 고민돼."

### 하루클릭 플로우
1. 청첩장 촬영
2. AI가 정보 자동 인식
3. 추천 금액 확인
4. 저장 누르면 끝

### 체감 포인트
`5분 걸리던 일이 3초 등록 경험`으로 바뀝니다.

---

## 4. 핵심 기능 (쉽게)

### 1) AI OCR 스캔
- 청첩장/부고장/영수증에서 핵심 정보 추출

### 2) 음성 입력
- "토요일 2시, 김OO 결혼식 10만원" 바로 등록

### 3) 금액 추천
- 관계 + 지역 + 행사 유형 기반 추천

### 4) 공유 캘린더 + 채팅
- 가족/팀 단위로 같이 보고 같이 결정

### 5) 리포트
- 월별 경조사비, 관계별 지출 흐름 확인

---

## 5. 왜 지금 필요한가

- 한국 경조사 관련 지출 규모는 매우 큼
- 하지만 관리 도구는 여전히 파편화
- 캘린더 앱은 일정만, 가계부 앱은 지출만 다룸

### 하루클릭 포지션
`일정 + 금액 + 관계`를 한 번에 다루는 경조사 특화 앱

---

## 6. 차별화 포인트

| 비교 항목 | 일반 캘린더 | 가계부 | 하루클릭 |
|---|---|---|---|
| 청첩장 자동 인식 | X | X | O |
| 음성으로 일정 등록 | X | X | O |
| 축의/부의금 추천 | X | X | O |
| 공유 캘린더 + 채팅 | 일부 | X | O |
| 주고받은 금액 추적 | X | X | O |

---

## 7. 수익 모델 (명확하게)

### B2C 구독
- Free: 월 제한 사용
- Pro: 무제한 OCR + 추천 + 통계
- Family: 가족 공유 특화

### B2B 확장
- 기업 총무팀용 경조사 관리
- OCR/추천 API 제공

### 핵심 논리
`반복 사용되는 생활 문제`라서 구독 유지율을 만들기 좋습니다.

---

## 8. 현재 상태와 현실적인 개선 과제

### 현재 강점
- 핵심 기능(인식, 추천, 공유, 채팅) 구현 완료
- Supabase RLS 기반 보안 적용

### 보완해야 할 점
- 추천 금액 근거를 사용자에게 더 투명하게 설명
- OCR 오인식 시 수정 UX 더 단순화
- 초기 온보딩에서 "첫 등록" 성공률 개선

---

## 10. KPI (집중할 숫자)

- 첫 주 활성 유지율 (D7 Retention)
- 첫 등록 완료율
- 추천 금액 채택률
- 월간 유료 전환율
- 팀/가족 공유 캘린더 생성률

### 원칙
큰 MAU 숫자보다 `재방문과 전환`을 먼저 증명합니다.

---

## 11. 리스크와 대응

### 리스크
- 금액 추천이 사용자 기대와 다를 수 있음
- 개인정보/민감정보 신뢰 이슈
- "한 번 쓰고 안 쓰는" 단발성 사용 가능성

### 대응
- 추천 근거 카드(관계/지역/행사) 노출
- PII 마스킹 + 권한 분리 강화
- 알림/리포트로 재방문 루프 설계
